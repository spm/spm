# SPM DCM M/EEG Regression Tests

Authored by Pranay Yadav in 2026

This folder contains code for testing neural mass models in DCM for M/EEG, for ERPs and
CSDs. The main suite is intentionally one discoverable MATLAB unittest class at the
SPM tests root folder:

```matlab
fullfile(spm('Dir'),'tests','test_regress_spm_dcm_meeg.m')
```

It exposes 117 parameterized tests: 9 models for tests 1, 2, 4, 6, and 7;
9 models x 3 integrators for test 3; and 9 models x 5 fixed datasets for
test 5. These tests are:
- Test 1: Check if model's steady-state Jacobian is stable
- Test 2: Check if model initializes correctly to its steady-state solution
- Test 3: Check if model can be integrated by a selection of integration schemes
- Test 4: Check if model can fit ERP generated by its own priors
- Test 5.1: Check if model can fit an ERP simulated by Jansen-Rit model
- Test 5.2: Check if model can fit a synthetic triphasic waveform
- Test 5.3: Check if model can fit ERP from sample data (source localized LFP)
- Test 5.4: Check if model can fit ERP from sample data (sensor-level IMG)
- Test 5.5: Check if model can fit ERP from sample data (sensor-level ECD)
- Test 6: Check if model can produce CSD under default priors
- Test 7: Check if model can fit CSD from sample LFP data
Tests 1, 2, 3 and 6 run forward simulations only, the remaining operate on data which
needs to be provided in spm-tests-data payload.

## Maintenance Scripts

Prepare fixtures explicitly when refreshing SPM test data:

```matlab
addpath(fullfile(spm('Dir'),'tests','DCM_Test_Suite'));
dts_prep01_make_test_fixtures();
```

This writes processed data for computing reference values under:

```text
fullfile(spm('Dir'),'tests','data','MEEG','DCM_Test_Suite','prep')
```

The reference values are then generated by running:

```matlab
dts_prep02_generate_reference_data(false);
```

Pass `true` to fit fixed-data references in parallel. Reference generation
covers the five fixed-data cases in test 5 and the CSD LFP fit in test 7.
Test 4 uses self-generated LFP data and checks fit quality by R2 threshold,
not stored F/Ep reference values. 

## Running Tests

SPM's explicit `test` selector only looks for test files directly under
`fullfile(spm('Dir'),'tests')`. The expected layout is:

```text
tests/test_regress_spm_dcm_meeg.m
tests/DCM_Test_Suite/
```

```matlab
results = spm_tests('class','regression','test','test_regress_spm_dcm_meeg','display',true);
```

Broad SPM regression discovery also works:

```matlab
results = spm_tests('class','regression','display',true);
```

## Description of tests

Tests 1-3 are model-level diagnostics:

- steady-state Jacobian stability;
- initialisation and baseline behaviour;
- forward integrator agreement, parameterized over `spm_int_L`,
  `spm_int_B`, and `spm_int_J`.

Test 4 generates one LFP dataset per model, fits that model back to its own
data, and checks finite outputs plus `self_lfp_min_r2`.

Test 5 fits every model to five prepared datasets. Fixture locations,
reference rows, and runtime tests all use the shared
`dts_meeg_fixed_data_cases` manifest:

- ERP-generated LFP;
- synthetic triphasic LFP;
- sample LFP;
- sample EEG IMG;
- sample EEG ECD.

For test 5, each fit checks finite `F`, `R2`, and `spm_vec(DCM.Ep)`, then
compares `F`, `R2`, and posterior parameters against shipped fixed-data
references. R2 is always compared with the reference value for the same
model-data combination. `F_margin` is the allowed absolute free-energy
drift in log-evidence units, and `Ep_margin` is the allowed maximum
absolute posterior-parameter drift. `strict_min_R2` and `basic_min_R2`
control the optional absolute R2 floor for strict and basic fits. Set either
value to 0 to disable that floor. Each concrete fit copies the selected value
into `cfg.thresholds.min_R2` before checking references. There are no ordering
tests in the default suite.

Test 6 runs prior CSD forward passes for single- and four-node LFP networks
and checks that `spm_csd_mtf` produces finite, non-zero spectra.

Test 7 fits an explicitly configured four-source CSD LFP model to SPM's
existing LFP fixture:

- `tests/data/MEEG/LFP_BG.mat`.

The CSD model configuration is defined in `dts_meeg_csd_lfp_template`.
The test compares `F`, `R2`, and posterior parameters with generated
references.
